# 第六天开发总结

## 完成的功能

### 1. 创建的文件

#### DTO 层
- `TransactionRequest.java` - 账单请求 DTO，包含完整的验证注解
- `TransactionResponse.java` - 账单响应 DTO
- `TransactionListResponse.java` - 账单列表响应 DTO，包含收入支出统计

#### Service 层
- `TransactionService.java` - 账单业务逻辑服务
  - `createTransaction()` - 创建账单并记录日志
  - `getTransactions()` - 获取账单列表，支持按年月筛选
  - `updateTransaction()` - 更新账单并记录日志
  - `deleteTransaction()` - 删除账单并记录日志
  - 自动计算收入支出总额

#### Controller 层
- `TransactionController.java` - 账单控制器
  - POST `/api/transactions` - 创建账单
  - GET `/api/transactions?year=2026&month=6` - 获取账单列表（支持月份筛选）
  - PUT `/api/transactions/{id}` - 更新账单
  - DELETE `/api/transactions/{id}` - 删除账单

#### 工具类
- `JsonUtil.java` - JSON 序列化工具，支持 Java 8 时间类型

### 2. 核心功能

#### 账单 CRUD
- ✅ 创建账单：支持收入/支出类型、类目、金额、日期、备注、图片URL
- ✅ 查询账单：支持按年月筛选，自动统计收入支出总额
- ✅ 更新账单：权限验证，只能修改自己的账单
- ✅ 删除账单：权限验证，只能删除自己的账单

#### 操作日志
- ✅ 所有增删改操作自动记录到 operation_log 表
- ✅ 记录操作前后的数据快照（JSON 格式）

#### 数据验证
- ✅ 账单类型必填
- ✅ 类目必填，最长50字符
- ✅ 金额必填，最小0.01，最多10位整数2位小数
- ✅ 日期必填
- ✅ 备注最长255字符
- ✅ 图片URL最长500字符

#### 权限控制
- ✅ 用户只能查看自己的账单
- ✅ 用户只能修改/删除自己的账单
- ✅ 未授权操作返回403错误

### 3. API 示例

#### 创建账单
```bash
POST /api/transactions
Headers: Authorization: Bearer {token}
Body:
{
  "type": "EXPENSE",
  "category": "餐饮美食",
  "amount": 50.00,
  "date": "2026-06-10",
  "remark": "午餐"
}
```

#### 获取账单列表（全部）
```bash
GET /api/transactions
Headers: Authorization: Bearer {token}
```

#### 获取账单列表（按月筛选）
```bash
GET /api/transactions?year=2026&month=6
Headers: Authorization: Bearer {token}
```

#### 更新账单
```bash
PUT /api/transactions/1
Headers: Authorization: Bearer {token}
Body:
{
  "type": "EXPENSE",
  "category": "餐饮美食",
  "amount": 60.00,
  "date": "2026-06-10",
  "remark": "午餐（修改）"
}
```

#### 删除账单
```bash
DELETE /api/transactions/1
Headers: Authorization: Bearer {token}
```

## 技术亮点

1. **事务管理**：使用 `@Transactional` 保证数据一致性
2. **操作审计**：所有增删改操作自动记录日志，便于追溯
3. **权限控制**：验证用户只能操作自己的数据
4. **数据验证**：使用 Bean Validation 进行参数校验
5. **统计功能**：查询列表时自动计算收入支出总额
6. **灵活查询**：支持按年月筛选或查询全部
7. **API 文档**：集成 Swagger，自动生成接口文档
