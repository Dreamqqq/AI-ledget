### 数据库表设计
```sql
-- 用户表
CREATE TABLE user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    phone VARCHAR(11) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    name VARCHAR(50),
    age INT,
    occupation VARCHAR(50),
    gender ENUM('MALE', 'FEMALE') NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 账单表
CREATE TABLE transaction (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    type ENUM('INCOME', 'EXPENSE') NOT NULL,
    category VARCHAR(50) NOT NULL,
    amount DECIMAL(10, 2) NOT NULL,
    date DATE NOT NULL,
    remark VARCHAR(255),
    image_url VARCHAR(500),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES user(id)
);

-- 操作日志表
CREATE TABLE operation_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    operation_type ENUM('CREATE', 'UPDATE', 'DELETE') NOT NULL,
    entity_type ENUM('TRANSACTION', 'USER') NOT NULL,
    entity_id BIGINT NOT NULL,
    old_value TEXT,
    new_value TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES user(id)
);
```

## 开发建议

1. **每天完成任务后提交 Git**（即使项目不是远程仓库，本地也要 commit）
2. **遇到问题先 Google/百度，解决不了再问 AI**
3. **优先保证核心功能，UI 美化放到最后**
4. **测试驱动开发：写完一个接口立即用 Postman 测试**
5. **Android 开发建议用模拟器测试，避免真机调试问题**

## 常见问题预案

| 问题                     | 解决方案                                      |
|--------------------------|-----------------------------------------------|
| JWT token 过期           | 前端拦截 401 错误，跳转登录页                 |
| OSS 上传失败             | 检查 AccessKey 权限、Bucket 公开设置          |
| GPT-5.4 识别不准确        | 优化 Prompt、添加示例、降级到手动输入         |
| Android 网络请求失败     | 检查 AndroidManifest.xml 权限、后端 CORS 配置 |
| 数据库连接失败           | 检查 MySQL 服务、用户名密码、时区配置         |
